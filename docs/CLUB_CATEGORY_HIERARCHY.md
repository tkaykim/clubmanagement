# 동아리 카테고리 계층 (대분류-중분류-소분류)

동아리는 **대분류 → 중분류 → 소분류** 3단계로 구분합니다.  
선택은 **대분류 필수**, 중·소분류는 선택입니다.

## DB 구조 (Supabase 적용됨)

- **category_major** (대분류): id, name, sort_order — 8개 시드 (댄스, 음악/밴드, 여행, 스포츠/운동, 문화/예술, 학술/전공, 봉사/친목, 기타)
- **category_middle** (중분류): id, major_id, name, sort_order — 위 표 기준 시드
- **category_minor** (소분류): id, middle_id, name, sort_order — 주요 중분류별 소분류 시드
- **clubs**: category_major_id (NOT NULL, 미지정 시 '기타'), category_middle_id (NULL 가능), category_minor_id (NULL 가능)

기존 **clubs.category**(단일 텍스트) 컬럼은 호환용으로 유지. 앱에서는 대·중·소 ID로 조회·표시하고, 필요 시 category는 대분류명 등으로 동기화할 수 있음.
